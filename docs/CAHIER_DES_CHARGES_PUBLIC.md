## 1. VUE D'ENSEMBLE

### 1.1 Contexte
Le site web public du CSAR est la vitrine institutionnelle permettant aux citoyens et partenaires de :
- Découvrir les missions et actions du CSAR
- Soumettre des demandes d'assistance
- Suivre l'actualité
- Accéder aux rapports publics
- Contacter l'organisation

### 1.2 Objectif Général
Créer un site web moderne, accessible et informatif pour :
- Informer le public sur les missions du CSAR
- Faciliter les demandes d'assistance citoyenne
- Assurer la transparence via les rapports SIM
- Communiquer les actualités et actions
- Renforcer la présence en ligne de l'organisation

### 1.3 Public Cible
- **Citoyens sénégalais** : Demandes d'assistance
- **Partenaires internationaux** : Collaboration et information
- **Médias** : Actualités et communiqués
- **Investisseurs** : Transparence et rapports
- **Grand public** : Information générale

### 4.12 Page Faire une Demande 📝

**Objectif** : Soumission de demandes d'assistance

**Types de Demandes** :
1. Assistance alimentaire
2. Aide matérielle
3. Information
4. Réclamation
5. Autre

**Formulaire** :
1. **Identification** :
   - Nom complet (obligatoire)
   - Prénom (obligatoire)
   - Email (obligatoire)
   - Téléphone (obligatoire)
   - CNI/Passeport (optionnel)

2. **Localisation** :
   - Région (sélection)
   - Département (sélection dynamique)
   - Commune (sélection dynamique)
   - Adresse complète
   - GPS (auto-détection ou manuel)

3. **Demande** :
   - Type de demande (sélection)
   - Objet/Titre
   - Description détaillée (500-2000 caractères)
   - Pièces justificatives (upload multiple, max 10 MB total)
   - Nombre de personnes concernées
   - Urgence (normale, moyenne, haute)

4. **Validation** :
   - Vérification des données
   - Consentement RGPD
   - CAPTCHA anti-spam
   - Acceptation conditions

**Workflow** :
1. Remplissage formulaire
2. Validation côté client (JavaScript)
3. Soumission → Validation serveur
4. Génération code de suivi unique (ex: DEM-2025-1234)
5. Enregistrement en base
6. Email de confirmation + code de suivi
7. SMS de confirmation (si numéro valide)
8. Notification admin
9. Page de succès avec instructions

**Fonctionnalités** :
- ✅ Sauvegarde automatique (brouillon)
- ✅ Validation progressive
- ✅ Upload drag & drop
- ✅ Preview fichiers
- ✅ Géolocalisation auto
- ✅ Sélection carte
- ✅ Estimation délai de traitement

## 6. DESIGN ET UX

### 6.1 Charte Graphique Publique

#### Couleurs
```css
/* Couleurs principales CSAR */
--csar-primary: #22c55e;      /* Vert CSAR */
--csar-secondary: #16a34a;    /* Vert foncé */
--csar-accent: #f59e0b;       /* Orange */
--csar-dark: #1f2937;         /* Gris foncé */
--csar-light: #f3f4f6;        /* Gris clair */
--csar-white: #ffffff;
--csar-black: #000000;

/* Couleurs Sénégal */
--senegal-green: #00853f;
--senegal-yellow: #fcd116;
--senegal-red: #e31b23;
```

#### Typographie
- **Titres** : Poppins, Montserrat (bold)
- **Corps** : Inter, Open Sans (regular)
- **Taille base** : 16px
- **Échelle** : 14px, 16px, 18px, 24px, 32px, 48px, 64px

#### Espacement
- Container : 1140px max-width
- Sections : 80px padding vertical
- Grilles : Gap 24px
- Cards : Padding 24px

### 6.2 Composants UI

#### Navigation
- **Desktop** : Menu horizontal sticky
- **Mobile** : Menu hamburger
- **Sections** :
  - Logo CSAR (gauche)
  - Menu principal (centre)
  - Bouton CTA "Faire une demande" (droite)
  - Sélecteur langue (FR/EN)

#### Hero Section
- **Hauteur** : 100vh (plein écran)
- **Background** : Image/vidéo + overlay
- **Contenu** : Centré
- **CTA** : 2 boutons (primaire + secondaire)
- **Scroll indicator** : Flèche animée

#### Cards
- **Style** : Moderne avec ombres
- **Hover** : Élévation + scale
- **Contenu** : Icône + titre + description
- **Link** : Toute la card cliquable

#### Footer
- **Sections** :
  - À propos + liens
  - Ressources
  - Contact
  - Newsletter
  - Réseaux sociaux
- **Copyright** : Centré en bas
- **Liens légaux** : Politique, Conditions, Mentions

### 6.3 Responsive Design

#### Mobile (<768px)
- Menu hamburger
- Sections empilées
- Grille 1 colonne
- Images full-width
- Touch-friendly (boutons 48px min)

#### Tablette (768-1024px)
- Menu complet ou hamburger
- Grilles 2 colonnes
- Optimisé touch

#### Desktop (>1024px)
- Menu horizontal
- Grilles 3-4 colonnes
- Effets hover
- Navigation souris
